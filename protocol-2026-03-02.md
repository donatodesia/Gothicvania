## Change 1: New Platforms (GameScene.js)

### Problem
The level map is 300 tiles wide but most of it was flat ground. The left side (x=1-40), a middle stretch (x=65-110), and the entire right side (x=185-290) had no platforms above ground level. It felt empty.

### What we did
We added a method called `addExtraPlatforms()` that runs right after the tilemap loads. It places platforms on **two layers at once**:

- **Collisions Layer** — tile index `2` (the invisible one-way collision tile). This is what the physics engine uses to decide "can the player stand here?"
- **Main Layer** — tiles `89, 90, 91, 92` in a repeating pattern. These are the visible ground-surface tiles so the player can actually *see* the platform.

### How it works step by step

1. We store a reference to the Main Layer (previously it was a throwaway variable):
   ```js
   // BEFORE (line 22 was):
   map.createLayer('Main Layer', [tileset, objects]);

   // AFTER:
   this.mainLayer = map.createLayer('Main Layer', [tileset, objects]);
   ```
   We need `this.mainLayer` so we can call `putTileAt()` on it later.

2. We call `this.addExtraPlatforms()` after the tilemap is set up but before the player is created (line 37).

3. The method itself is a list of `{ x, y, w }` objects. Each one describes a platform:
   - `x` = the tile column where the platform starts
   - `y` = the tile row (0 = top of map, 12 = ground level)
   - `w` = how many tiles wide

4. For each tile in each platform, we do two things:
   ```js
   // Place a one-way collision tile (invisible)
   const tile = this.collisionLayer.putTileAt(2, x + i, y);
   tile.setCollision(false, false, true, false);
   //                 left   right  up    down
   // "true" on up means: collide when player is ABOVE the tile (landing on it)
   // "false" everywhere else means: player passes through from sides and below

   // Place a visible surface tile so the player can see it
   this.mainLayer.putTileAt(surfaceTiles[i % surfaceTiles.length], x + i, y);
   // surfaceTiles = [89, 90, 91, 92] — cycles through them for visual variety
   ```

## Change 2: Drop-Through Mechanic (Player.js + GameScene.js)

### Problem
One-way platforms let you jump through them from below and land on top — but once you're standing on one, you're stuck. There was no way to go back down without walking off the edge.

### What we did
We added a **Down + Space** input that lets the player fall through any one-way platform they're standing on.

### How it works step by step

**In Player.js (the player entity):**

1. Added a new state flag in the constructor:
   ```js
   this.droppingThrough = false;
   ```
   This flag tells the physics system "ignore one-way platform collisions right now."

2. Registered the Space key as an input (line 106):
   ```js
   this.keySpace = scene.input.keyboard.addKey(Phaser.Input.Keyboard.KeyCodes.SPACE);
   ```

3. Added the drop-through check in `update()` (lines 197-204):
   ```js
   if (this.cursors.down.isDown && Phaser.Input.Keyboard.JustDown(this.keySpace) && onFloor) {
     this.droppingThrough = true;   // tell physics to skip one-way collisions
     this.y += 2;                   // nudge player down 2px so they clear the tile
     this.scene.time.delayedCall(200, () => {
       this.droppingThrough = false; // re-enable collisions after 200ms
     });
   }
   ```
   Key details:
   - `this.cursors.down.isDown` — Down arrow must be **held**
   - `Phaser.Input.Keyboard.JustDown(this.keySpace)` — Space must be **freshly pressed** (not held)
   - `onFloor` — player must be standing on something (can't drop-through mid-air)
   - `this.y += 2` — without this nudge, the player would still be overlapping the tile and the collision would catch them again next frame
   - The 200ms timer gives the player enough time to fall past the platform before collisions turn back on

**In GameScene.js (the collider):**

4. Changed the collider to use a **process callback** (lines 44-47):
   ```js
   // BEFORE:
   this.physics.add.collider(this.player, this.collisionLayer);

   // AFTER:
   this.physics.add.collider(this.player, this.collisionLayer, null, (player, tile) => {
     if (player.droppingThrough && tile.index === 2) return false;
     return true;
   });
   ```
   The 4th argument is a "process callback." Phaser calls it every frame for every tile the player overlaps. If it returns `false`, the collision is **skipped**. So:
   - If the player has `droppingThrough = true` AND the tile is a one-way platform (index 2) → skip collision → player falls through
   - Otherwise → normal collision

## Change 3: Jump Requires Fresh Press (Player.js)

### Problem
The jump used `this.cursors.up.isDown` which is `true` on **every frame** while the key is held down. This meant: if you held Up and landed on a platform, you'd immediately jump again without meaning to. The player effectively "bounced" uncontrollably.

### What we did
Changed one line (line 207):
```js
// BEFORE:
if (this.cursors.up.isDown && onFloor) {

// AFTER:
if (Phaser.Input.Keyboard.JustDown(this.cursors.up) && onFloor) {
```

### Why this works
- `isDown` = true every frame the key is held
- `JustDown()` = true only on the **first frame** the key goes from "not pressed" to "pressed"

So now you must release Up and press it again to jump again. This is standard platformer behavior.

## Key Concepts Used

### Phaser Tilemaps — Two-Layer Pattern
Our game uses separate layers for **visuals** (Main Layer) and **physics** (Collisions Layer). The Collisions Layer is invisible (`setVisible(false)`). This lets us control collision behavior independently from how things look. When we add a platform, we must add tiles to **both** layers.

### One-Way Platforms (setCollision)
```js
tile.setCollision(left, right, up, down)
```
Each parameter is a boolean. Setting only `up` to `true` means the player collides when falling onto the tile from above, but passes through from all other directions.

### Process Callbacks
The 4th argument to `physics.add.collider()` is called before each collision is resolved. Returning `false` cancels that specific collision. This is how we selectively disable collisions at runtime (for drop-through) without removing the tile.

### JustDown vs isDown
- `key.isDown` — true on **every frame** while held. Good for continuous actions (movement).
- `Phaser.Input.Keyboard.JustDown(key)` — true only on the **first frame** of a press. Good for discrete actions (jump, drop-through).
